# Message Filter — 私信留言价值过滤器

帮内容创作者评估粉丝私信和留言，按多维标准判断哪些值得投入时间深入交流，并通过追问机制探明留言者的真实诉求，最终给出「深入交流 / 简短回复 / 无需回复」的建议。

> ⚠️ **核心原则**：你不是客服，你的时间是创作资本。每一条回复都是在用创作时间投票。这个工作流帮你把时间票投给真正值得的对话。

---

## Step 1：接收留言内容

用户提供粉丝留言的完整内容（原文照搬，不做任何修改）。

若用户只提供了摘要或转述（而非原文），提醒用户尽量提供原文以获得最准确的评估，并等待用户补充。若用户坚持使用摘要（回复「就这样」「没关系」「用这个判断吧」等），则按现有信息继续进入 Step 2，但在评分中将「留言者投入度」降低一个档位（非原文导致评估精度损失，保守处理）。

### 1.1 记录元信息

同时记录：
- **留言渠道**：私信 / 评论区 / 邮箱 / 其他（影响回复的郑重程度）
- **留言者身份**：老粉 / 新粉 / 同行 / 路人（如能判断）

---

## Step 2：六维价值评估

对留言内容按以下六个维度逐条打分，每个维度 1-5 分（内部分析，不逐条输出给用户）。

### 六个评估维度

| 维度 | 说明 | 1 分 | 5 分 |
|------|------|------|------|
| **清晰度** | 问题是否明确、可回答 | 「你好」「在吗」「大佬」 | 具体的、有上下文的问题 |
| **深度** | 是否超出 FAQ/百度可搜 | 「怎么入门 xxx」 | 有行业洞察、有思考拐点的问题 |
| **领域相关性** | 与你的创作方向是否匹配 | 与你领域完全无关 | 你恰好是这个问题的最佳回答者 |
| **留言者投入度** | 对方花了多少心思 | 一句话、拼凑 | 写了背景、尝试过什么、有逻辑 |
| **独特性** | 是否高频重复问题 | 你每天被问了 50 遍的 | 你第一次被问到的角度 |
| **行动价值** | 交流后是否可能产生价值 | 纯消耗型提问 | 可能带来合作、新选题、付费咨询 |

### 2.2 综合评分

加权计算（权重可在实践中调整）：

| 维度 | 权重 |
|------|------|
| 清晰度 | 15% |
| 深度 | 25% |
| 领域相关性 | 20% |
| 留言者投入度 | 15% |
| 独特性 | 10% |
| 行动价值 | 15% |

综合得分 ≥ 3.5 → 进入 Step 2.3（确认分流）后执行 Step 3
综合得分 2.0-3.4 → 进入 Step 2.3（确认分流）后执行 Step 4
综合得分 < 2.0 → 进入 Step 2.3（确认分流）后执行 Step 5

### 2.3 展示评估摘要并确认

完成评分后，向用户输出简短摘要，等待确认后再执行对应分流步骤。

> 💡 用户可在 Step 1 时声明「跳过确认，直接出结果」，则此步骤省略，评分后直接进入对应 Step。

输出格式：

```
📊 评估结果：[N.N 分] — [追问 / 简短回复 / 不回复]

- 亮点：[该留言最强的 1 个维度，一句话]
- 短板：[该留言最弱的 1 个维度，一句话]

按评分将进入「[分流方向]」流程，确认？（Y / 调整 / 跳过确认）
```

用户回应处理：
- Y / 确认 → 执行对应 Step（3/4/5）
- 调整 → 询问用户认为应该修正哪个维度及原因，修正后重新判断分流方向，再次输出摘要确认（此循环最多 2 轮，第 2 轮后直接执行）
- 跳过确认 → 直接执行对应 Step（后续可一直跳过，用户已表示信任判断）

---

## Step 3：追问 — 探明真实诉求

> 此步骤仅对 Step 2 评分 ≥ 3.5 且用户确认后执行。

### 3.1 生成追问问题

根据留言内容，生成 1-3 个追问问题，目的是：

- **探明深度**：对方的真实目的是什么？（解决具体问题 / 获取资源 / 建立联系 / 白嫖？）
- **检验诚意**：对方愿意为这个问题投入多少？（「你能帮我看看吗」→ 「你目前尝试过哪些方案？遇到的具体卡点是什么？」）
- **锚定边界**：这个问题值得花多久？（5 分钟回答 vs 需要深度讨论）

### 3.2 追问格式

输出给用户回复留言者的追问：

```
[称呼]，你提的问题很好，为了能给你更有价值的回应，我想先了解几个点：

1. [追问 1 — 探背景]
2. [追问 2 — 探动机/诚意]
3. [追问 3 — 探预期（可选）]

这些信息能帮我判断是否能真正帮到你。
```

### 3.3 追问后分流

用户将追问发给对方、收到回复后，再次触发本技能（提供追问后的对话），进入第二轮评估：

- 对方认真回复 → **深入交流**（投入时间，展开讨论）
- 对方敷衍/不回复 → 自动降级为 **无需进一步跟进**
- 对方表露付费意向 → 引导至付费咨询渠道

---

## Step 4：简短回复分流

> 此步骤对评分 2.0-3.4 且用户确认后执行。

生成一条简短、礼貌、但关闭深入对话通道的回复：

```
[称呼]，感谢你的留言。关于 [主题]，[1-2 句核心回应/已有资源的指向]。这个问题我暂时没法展开细聊，希望能帮到你。
```

可选操作：
- 附上相关文章的链接（如果已有现成内容可以回答）
- 引导至公开的 FAQ 或评论区

---

## Step 5：无需回复

> 此步骤对评分 < 2.0 且用户确认后执行。

不做任何输出。仅向用户反馈：

```
🔕 建议不回复。
原因：[简述，如「纯打招呼，无具体诉求」「问题过于宽泛，超出可回复范围」]
```

## 进阶策略：组合过滤链

此技能的完整过滤链设计为三层，Step 2-5 是核心层（AI 评估层）。用户可选择性启用另外两层：

| 层级 | 机制 | 效果 |
|------|------|------|
| **第一层：渠道过滤** | 只接受邮箱作为深度交流入口，关闭私信区 | 过滤掉「随便问问」的人 — 愿意写邮件的人已经完成了第一次自我筛选 |
| **第二层：AI 评估层** | 本技能的六维评估 + 追问机制 | 从写邮件的留言中进一步筛选出高价值对话 |
| **第三层：付费门槛** | 设置付费咨询价位，深入交流前引导至付费通道 | 二次过滤 — 真正有需求且认可你价值的人会买单 |

建议用法：
- **初期**（粉丝量不大）：仅使用本技能的 AI 评估层，保持与粉丝的互动感
- **中期**（私信开始占据大量时间）：开启邮箱策略，缩短私信回复
- **后期**（需要把时间变现）：三层全开，邮箱 + AI 评估 + 付费门槛

## 边界情况

| 情况 | 处理 |
|------|------|
| 留言是纯夸奖/感谢（「你说得太好了」） | 评分通常 < 2.0，建议不回复或用预设的简短感谢模板（如已有） |
| 留言是负面/攻击性内容 | 直接标记为「无需回复」，不消耗评估资源 |
| 留言是商业合作/广告邀约 | 不按六维评分，跳过 Step 2，直接提示用户自行判断商业价值 |
| 留言内容太短无法评估（「在吗」「你好」） | 清晰度直接打 1 分，综合分通常 < 2.0 |
| 同一人多次留言 | 参考历史评估记录。若此前已被标记为低价值，新留言仍低价值时直接跳过；若新留言质量明显提升，独立评估 |
| 用户同时提交多条留言 | 逐条评估，最后汇总优先级排序 |
| 留言包含敏感/违法内容 | 不做评估，建议用户直接举报或拉黑 |
| 评论区的公开留言 | 评估逻辑相同，但回复策略需考虑「公开可见」的因素——简短回复比完全不回更好 |

---

## 禁止清单

- ❌ 禁止对低分留言生成「深入回复」——评分是硬约束，不要因为「万一有价值」而破例
- ❌ 禁止在 Step 2 评估完成前直接跳到回复生成
- ❌ 禁止让 AI 代替用户做出最终沟通决定——AI 是过滤器，用户是决策者
- ❌ 禁止用模板化的客服语气生成回复——保留创作者的个人风格
- ❌ 禁止在追问环节（Step 3）输出攻击性或审判性的问题——追问的目的是探明，不是质问
