# 为 AI Playbook 搭建私信留言价值过滤器

> 从一次粉丝互动出发，将「内容创作者如何平衡创作时间与私信回复」的讨论，沉淀为一个可复用的 AI 技能——六维评估体系自动判断留言是否值得投入时间，附带追问机制和三层过滤链。

## 背景

一位微信公众号粉丝加了作者的微信，问了一个元问题：**你是如何平衡创作时间投入和私信留言回复的？** 这个问题本身就很妙——它在问一个内容创作者如何管理注意力资源。

讨论中碰撞出几条策略：告诉 AI 什么类型的留言有意义（多维度过滤）、设置付费咨询门槛、只留邮箱作为深度交流入口（愿意郑重其事写邮件的人已经完成了第一次自我筛选）。归根结底，需要一个 **workflow** 来追问留言者的真实诉求，判断是否值得继续对话。

这就是本次对话的起点——把这个思路结构化，变成 AI Playbook 里一个可复用的资产。

## 过程

### 想法分类：该沉淀成什么？

用 `idea-classifier` 技能判断这个想法的归属。八个候选类型中，最接近的是「技能」和「提示词」：提示词方案适合单轮评估（把留言贴进去，AI 打分，出结论），但用户明确提到了「需要一个 workflow」——多步骤、有分支判断、追问后分流。这超出了提示词的能力边界，落在了「技能」的定义区间。

最终确定为 **技能**，路径 `skills/message-filter/`。

### 技能生成：六维评估 + 三层过滤链

技能的骨架是五步工作流：

1. **接收留言**：原文照搬，记录渠道和留言者身份
2. **六维评估**：清晰度、深度、领域相关性、留言者投入度、独特性、行动价值——各 1-5 分，加权计算后定级
3. **追问（高分）**：探明对方真实目的、检验诚意、锚定时间边界
4. **简短回复（中分）**：礼貌应答但关闭深入对话的通道
5. **丢弃（低分）**：建议不回复，给出一句话原因

在这个核心流程之外，还设计了**三层组合过滤链**：渠道过滤（邮箱入口）→ AI 评估层（本技能）→ 付费门槛，创作者可以按所处阶段选择性启用——初期只用 AI 层保持互动感，中期开邮箱策略，后期三层全开来变现。

### 质量审计：自己写的 Skill 也有盲区

用 `skills-audit` 审计后发现，六个维度整体不错（D1 触发器、D5 边界覆盖、D6 输出格式都通过），但有两个警告：

- **D2（步骤 I/O 清晰度）**：Step 1 说「若用户只给了摘要，提醒用户提供原文」——但提醒之后是等还是走？没说。
- **D3（分支路径完整性）**：评估完直接跳到输出，中间没有用户确认节点。AI 可能在深度或领域相关性上误判，用户却没有原地纠正的入口。

综合评级「达标」（3/5，无必须修复项），这两个警告不致命但会影响实际使用体验。

### 修复完善：补上缺失的路径

两处修复：

1. **Step 1 非原文路径**：明确为「先等待补充 → 用户坚持则继续但投入度降档处理」，不留空白。
2. **Step 2.3 确认节点（新增）**：评估完成后先向用户展示摘要（评分 + 亮点/短板 + 分流方向），提供三条路径：确认执行 / 调整评分（最多 2 轮循环）/ 跳过确认。同时支持用户在 Step 1 声明「跳过确认」来永久省略此节点。

修复后复审计：6/6 通过，🏆 优秀。

## 关键决策

| 决策 | 原因 |
|------|------|
| 分类选「技能」而非「提示词」 | 需要分步 + 分支 + 追问后分流，提示词只能承载单轮评估，装不下完整 workflow |
| 六维评分用加权而非等权 | 深度（25%）和领域相关性（20%）比清晰度（15%）更能区分「值得深聊」和「FAQ 可答」的留言 |
| 设计三层过滤链但渐进启用 | 不把付费门槛写进默认流程——粉丝量少的创作者需要互动感，一刀切反而有害 |
| 审计后新增 Step 2.3 确认节点 | 禁止清单写了「AI 不能代替用户决策」，但原流程里没有给用户决策的入口，这个矛盾只能靠确认节点解决 |
| 调整循环设 2 轮硬上限 | 避免用户在评分上反复纠结，破坏过滤器的「省时间」初衷 |

## 思考与备注

- **审计的价值**：写技能时很容易带入「理想执行路径」的假设——用户一定给原文、AI 一定评对分、用户一定同意。但现实中用户可能只扔个摘要过来，AI 可能误判领域相关性，用户可能想推翻 AI 的判断。审计把这些隐性的「万一」逼出来，转化成显性的分支路径。
- **轻量确认的平衡**：Step 2.3 的确认节点不能太重——如果每次都让用户看六维明细再确认，过滤器就从省时间变成费时间。所以摘要只展示最强和最弱的维度各一个，同时提供了「永久跳过」的后门。
